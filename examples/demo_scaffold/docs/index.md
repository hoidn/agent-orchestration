# Workspace Index

Start here.

## Key Files

- `state/task.md`: canonical task description injected at runtime
- `docs/dev_guidelines.md`: local engineering rules
- `docs/plans/templates/`: optional planning and review aids
- `src_py/`: visible source/reference implementation area
- `rust/`: visible target project area
- `artifacts/`: execution, check, and review outputs
- `state/`: workflow state and task artifacts

## Expectations

- Read the task before changing code.
- Use the plan and review templates if they help.
- Keep notes and reports in the expected artifact locations.
- Derive visible checks from the task and repository contents.
